자원 예약 시스템

Smart Multi-Resource Reservation IoT System

본 프로젝트는 2대의 라즈베리 파이(Pi A와 Pi B)를 유기적으로 연동하여 회의실, 학과 PC, 공유기 등 다양한 공용 자원을 실시간 제어하고, 물리 액추에이터(LED, LCD, Buzzer)를 통해 상태 피드백을 동기화하는 스마트 자원 관리 시스템입니다.

1. 개발 배경 및 필요성

03 / 11
💡 공용 자원의 고질적 문제인 무단 점유와 예약 노쇼(No-Show)를 시스템적 매커니즘으로 해결

🚨 기존 공용 자원 관리의 한계

  • 비인가 사용 및 무단 점유: 예약 정보가 웹에 실시간 연동되지 않고 현장 확인만 가능하여 타인의 예약을 임의 취소하거나 가로채는 마찰 발생
  • 예약 No-Show 독점 문제: 예약만 올려두고 방문 및 점유를 하지 않아 자원이 불필요하게 대기 상태로 방치되고 타인의 이용 권리가 침해됨
  • 물리 피드백 결여: 대시보드를 계속 쳐다보지 않으면 현장 부품의 예약이나 잔여 시간 경과 여부를 주변 사람들이 눈치채기 어려움

✅ 스마트 예약 시스템의 해결 대안

  • 웹 실시간 대시보드: 5초 단위의 브라우저 비동기 HTTP 폴링을 이용하여 원격에서도 모든 기기의 예약 상황을 동기화하여 시각화
  • PIN 기반 도용 차단: 로그인 절차 대신 예약 시 설정하는 4자리 PIN을 대조해 본인 인증을 수행하여 타인의 해제와 취소를 차단
  • 만료 스케줄러 도입: 백그라운드 스레드가 예약 후 노쇼 시(1분 미사용) 강제 폭파하고 사용 한도(3분) 초과 시 자동 반납 처리

라즈베리 파이 4 (2대)

  • Pi A: Apache 웹서버 구동, CGI 소켓 통신을 담당하는 웹 게이트웨이

  • 노랑: 예약 완료(Reserved)
  • 빨강: 사용 중(In Use)

포커스 기기 문자 디스플레이

  • 가변저항(Vo 핀) 조절을 통해 밝기 대비(Contrast) 하드웨어 제어

  • 사용 시작: 도-미-솔 상승음 출력
  • 예약 만료/반납: 솔-미-도 하강음 출력

4. 전체 시스템 구성도

06 / 11

📟 Pi A (Web Gateway)

사용자 브라우저 접속 수용 및 신호 릴레이 게이트웨이

  • Apache2 호스팅: 포트 80에서 대시보드 리소스(HTML, app.js) 서빙
  • ExecCGI 모듈: 웹 브라우저의 비동기 상태 변경 요청 접수
  • CGI Script (Python): 전달받은 명령을 해석하여 Pi B로 저지연 TCP 소켓 중계 송신

🔌 이원화 통신 파이프라인

지연시간 최소화와 서버 부하 방지를 위한 하이브리드 결합

  • 제어 명령 (저지연): 웹 요청 발생 시 Pi A CGI와 Pi B TCP 소켓 리스너 간의 Raw TCP 통신(포트 50007)을 사용해 즉시 하드웨어 구동
  • 상태 조회 (REST API): 사용자 대시보드 갱신을 위해 Pi B의 Flask API(포트 5000)를 5초마다 비동기 호출

⚙️ Pi B (Hardware Core)

자원 예약 로직 처리 및 물리 액추에이터 제어 본부

  • Flask API Server: 예약 데이터 처리 및 잔여 카운트다운 시간 계산 결과 서빙
  • Socket Listener Thread: 백그라운드 데몬 스레드로 기동하여 Pi A의 제어 요청 상시 대기
  • 데이터 잠금 (Mutex Lock): 다중 스레드(Flask/Socket)의 DB 동시 접근 경쟁 상태(Race Condition) 방지
  • JSON DB: 예약 상태를 data.json에 영속 보관

5. 전체 시스템 연동 흐름도

07 / 11
1. 웹 예약/제어 요청
사용자가 브라우저 대시보드(index.html)에서 예약 버튼을 누르고 본인 확인용 4자리 비밀번호(PIN) 설정 및 전달
2. CGI 소켓 송신
Pi A Apache 서버의 CGI 스크립트(update_status.py)가 비동기 요청을 가로채 Pi B의 포트 50007로 소켓 패킷 즉각 릴레이
3. 소켓 수신 & Mutex Lock
Pi B 백그라운드 소켓 리스너 스레드가 이를 감지, Flask API 조회 스레드와 데이터 충돌을 방지하기 위해 Mutex Lock 획득 후 data.json 갱신
4. 하드웨어 물리 제어
갱신된 상태에 따라 GPIO 모듈 동작 (LED 상태등 변환, 피에조 부저 도-미-솔 연주, Focused LCD 텍스트 갱신) 후 락 해제
5. 백그라운드 스케줄러
Pi B 내부의 데몬 스레드가 1초마다 루프를 돌며 예약 대기시간 초과(노쇼 1분) 및 사용 한도(3분) 초과 기기를 대기(Idle) 상태로 강제 복구
6. 실시간 웹 화면 동기화
웹 프론트엔드가 5초마다 비동기 AJAX를 통해 Flask API를 조회하여 갱신된 점유 정보와 남은 시간 타이머를 대시보드에 렌더링

6. 주요 기술 명세 및 동기화 구현

08 / 11
📡

CGI 연동 기술

Apache & ExecCGI
  • 웹 클라이언트(JS)와 백엔드 서버 간의 데이터 게이트웨이 스크립트 동작
  • 웹 요청 환경변수를 파싱해 서버 소켓 연결로 이어주는 동적 커넥터 역할 수행
🔌

저지연 TCP 소켓

포트 50007 제어 릴레이
  • HTTP 프로토콜의 헤더 오버헤드를 생략하고 원시 TCP 커넥션을 맺어 제어 지연시간 최소화
  • Pi A CGI에서 Pi B 소켓 리스너로 실시간 JSON 문자열 패킷 즉각 송신
🧵

멀티스레딩

Daemon Threads 구현
  • 소켓 리스너 스레드와 타이머 스케줄러 스레드가 메인 API(Flask)와 별개로 병렬 실행
  • 데몬 모드로 작동하여 메인 시스템 종료 시 부작용 없이 동시 회수 및 종료
🔒

뮤텍스 락 (Mutex Lock)

상호 배제 (Lock)
  • threading.Lock() 객체를 생성하여 데이터 및 물리 기기 동시 제어 충돌 차단
  • 소켓 쓰기 동작 중 불완전한 상태 정보를 Flask API가 읽지 못하게 크리티컬 섹션 완벽 보호
