# [프로젝트 제안서] 웹서버를 통한 공용 공간 및 물품 실시간 사용 관리 시스템

본 제안서는 두 대의 라즈베리 파이를 활용하여 웹 기반의 사용자 인터페이스와 물리 하드웨어가 동기화되는 스마트 예약 시스템을 구축함으로써, 여러 공용 공간과 기자재를 실시간으로 투명하게 관리하기 위한 연구 계획을 담고 있습니다.

---

## 1. 과제(연구) 목표 (Objectives)

본 과제의 목표는 교내 또는 학과 사무실 등 공동체 내에 존재하는 다양한 공용 공간(회의실, 스터디룸 등) 및 공용 물품(학과 컴퓨터, 공용 스테이플러, 라즈베리파이 테스트 키트, 공유 공유기 등)의 질서 있는 사용을 시스템적으로 뒷받침하여, 무단 점유 및 비인가 사용 등으로 발생하는 **사용자 간의 분쟁과 갈등을 미연에 방지**하는 것입니다.

*   **실시간 점유 정보의 시각화 및 투명성 제공**: 사용자가 공간이나 물품을 찾기 전 웹 페이지를 통해 현재 사용 현황을 확인하고, 예약 및 사용 등록을 비동기 처리함으로써 자원의 투명한 흐름을 제공합니다.
*   **간편 사용자 식별을 통한 소유권 침해 방지**: 로그인/가입 절차 없이, 예약 시 입력한 4자리 간편 비밀번호(PIN)를 통해 사용자를 식별함으로써 타인이 예약을 가로채 사용하거나 강제 취소하지 못하게 차단합니다.
*   **자원 회전율 극대화 및 공정성 확보**: 예약 후 장시간 방치되는 노쇼(No-show)를 스케줄러를 통해 강제 취소하고, 기기별 고정된 이용 시간이 지나면 자동으로 대기 상태로 반환시켜 독점을 방지하고 공정한 이용을 분담합니다.

---

## 2. 연구 내용 (Research Contents)

### 2.1. 하드웨어 구성 및 사용 부품

본 과제는 다음과 같은 하드웨어 부품들을 유기적으로 회로 구성하여 제어합니다.
*   **라즈베리파이 (Raspberry Pi) 2개**:
    *   **Pi A (Web Gateway)**: 외부 웹 접속을 서빙하고 CGI 기반 소켓 클라이언트를 통해 변경 명령을 Pi B로 안전하게 중계하는 역할을 수행합니다.
    *   **Pi B (Hardware Control Server)**: 소켓 통신을 통해 신호를 수신받아 물리 하드웨어 상태를 갱신하고, Flask API 제공 및 실시간 상태를 디렉토리 내 JSON 파일 DB로 보관합니다.
*   **LED 3개**: 대기(초록), 예약(노랑), 사용(빨강)의 3가지 상태 변화를 물리 공간에 가시적으로 표시하는 신호등 역할을 합니다.
*   **피에조 패시브 부저 (Passive Buzzer) 1개**: 예약에서 사용으로 전환 시 상승음(도-미-솔), 사용에서 반납(대기) 시 하강음(솔-미-도)의 3음계 멜로디를 출력해 상태 변이를 청각적으로 알립니다.
*   **문자형 LCD (16x2 Character LCD) 1개**: 하드웨어 모니터링 대상으로 지정된 자원의 영문 이름과 현재 상태(Available, Reserved, In Use)를 텍스트로 실시간 출력합니다.
*   **가변저항 (Potentiometer) 1개**: LCD의 Vo(화면 대비/밝기) 제어 핀에 연결되어 글자의 밝기 대비(Contrast)를 물리적으로 섬세하게 조절함으로써 텍스트의 시각적 가시성을 극대화합니다.

### 2.2. 소프트웨어 적용 기술 및 메커니즘

*   **웹 서버 (Apache)**:
    *   Pi A에 Apache Web Server를 설치하여 대시보드 화면(`index.html`, `app.js`)을 호스팅합니다.
    *   Apache의 ExecCGI 모듈을 통해 파이썬 스크립트(`update_status.py`)를 CGI 게이트웨이로 동작시켜 비동기 요청을 안정적으로 소켓 데이터로 변환합니다.
*   **저지연 TCP 소켓 통신**:
    *   Pi A의 CGI와 Pi B의 백그라운드 수신 리스너 간에 TCP Raw Socket 통신(포트 50007)을 구축합니다.
    *   웹에서 전달받은 `{"action": "change_state", "item_id": "...", "state": "...", "pin": "..."}` 구조의 JSON 데이터를 소켓을 통해 저지연으로 Pi B에 중계하여 하드웨어를 제어합니다.
*   **Flask REST API & 스레드 안전성**:
    *   Pi B는 Flask API(포트 5000)를 상시 기동하여 웹 대시보드의 5초 주기 상태 조회를 처리합니다.
    *   조회 스레드(Flask)와 쓰기 스레드(Socket) 간의 경쟁 상태(Race Condition)를 완벽히 막기 위해 파이썬 `threading.Lock` 상호 배제(Mutex) 기법을 사용하여 데이터 오염을 차단합니다.
*   **백그라운드 자동 만료 타이머 스케줄러**:
    *   Pi B 내부의 독립된 백그라운드 스레드가 1초마다 모든 기기의 상태를 검사합니다.
    *   예약 후 '예약 유효 대기 시간'을 넘길 때까지 사용으로 전환하지 않거나, 사용 시작 후 '고정 사용 한도 시간'이 지나면 즉시 강제로 `idle` 상태로 갱신하여 DB 및 하드웨어를 초기화합니다.
*   **경량 JSON DB & 안전 예외 처리**:
    *   `data.json` 파일을 사용하여 상태 데이터를 보존하고, 파일 쓰기 실패 시 예외를 상위로 전파(Exception Propagation)하여 사용자가 상태 전이 실패 원인을 즉각 인지하도록 설계했습니다.

## 3. 기대 효과 (Expected Effects)

*   **무단 점유 및 분쟁 방지**: 
    공용 공간이나 기자재를 무단으로 가로채거나 비인가자가 점유하는 행위를 PIN 인증 방식을 통해 완전히 예단하여 공동체 내의 분쟁과 갈등 요소를 사전에 차단합니다.
*   **공용 물품 관리의 대중화 및 효율성 제고**:
    회의실과 같은 커다란 공간뿐만 아니라 학과 사무실 PC, 공용 스테이플러 등 예약제를 도입하기엔 다소 사소하다고 여겨졌던 물품들에도 간편한 실시간 대시보드를 연동함으로써 분실을 막고 이용률을 추적하는 등 공용 인프라 운영의 합리성을 극대화합니다.
*   **예약 부재(No-show) 방지를 통한 자원 낭비 최소화**:
    예약을 해두고 나타나지 않는 비합리적인 예약 독점(No-show)을 스케줄러 스레드가 자동으로 감지하여 강제 취소하므로, 기회를 대기하는 다른 사람들에게 실시간으로 자원을 공정하게 배분합니다.
*   **자연스러운 이용 질서 확립**:
    소리(부저), 빛(LED), 화면(LCD)이라는 3대 물리 피드백 요소를 통해 주위 사람들에게 공간/물건의 반납 기한 및 점유 상태를 시각·청각적으로 환기하여, 기기의 오랫동안 독점하던 습관을 버리고 이용 시간 준수를 시스템적으로 습관화시킬 수 있습니다.
